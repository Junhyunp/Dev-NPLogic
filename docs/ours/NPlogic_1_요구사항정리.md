# NPlogic (NP로직) - 요구사항 정리

## 핵심 요구사항 정리

### 프로젝트 개요
- **프로젝트명**: NP 로직 (등기부등본 자동화 시스템)
- **프로젝트 유형**: C# 데스크톱 애플리케이션 (Windows)
- **목적**: 부실채권 인수 회사의 등기부등본 검색 및 데이터 입력 업무 자동화로 업무 효율성 향상
- **주요 사용자**: 회계사(평가자), PM, 관리자 (초기 1명 → 확장 멀티 유저)

### 핵심 기능
1. **등기부등본 PDF 대량 업로드 및 자동 데이터 추출** (OCR 시스템 통합)
2. **엑셀 데이터 디스크 업로드 및 등기부등본 데이터 자동 매칭**
3. **추출 데이터 실시간 수정, 병합, 삭제 기능**
4. **복잡한 계산 수식 자동 적용 및 여러 시트 통합 엑셀 출력**
5. **지도 API 연동 물건 위치 확인** (디스코 맵)
6. **유사 물건 추천 및 통계 분석**
7. **역할 기반 권한 관리** (PM, 평가자, 관리자)
8. **물건 관리 시스템** (검색, 필터, 정렬, 페이지네이션)

### 기술 요구사항
- **플랫폼**: C# Windows 데스크톱 애플리케이션 (WPF/WinForms)
- **클라우드 기반**: Supabase (PostgreSQL, Auth, Storage)
- **클라이언트 제공 자료**:
  - `Auction-Certificate.zip`: OCR 코드 실행용 데이터 및 결과 파일
  - `비핵심 프로그램화_v6-1.xlsx`: 화면 구성 및 데이터 매핑 자료 (매우 상세함)
  - Python OCR 코드 (C# 환경에 통합 필요)
  - HTML 지도 코드 (WebView 통합 필요)
  - 유사 물건 추천 알고리즘
- **대량 처리**: 500~3,000개 등기부등본 PDF 처리 최적화 필요
- **디자인 요구사항**:
  - 차분하고 딱딱한 엑셀 같은 느낌
  - 베이스 색상: 딥네이비
  - 표 색상: 살짝 밝은 블루그레이

### 일정 및 예산
- **개발 기간**: 7주 (약 2개월)
- **개발 비용**: 1,300만원 (부가세 별도)
- **계약 조건**:
  - 계약금: 100만원 (작업 시작 전)
  - 중도금: 600만원 (4주차 산출물 확인 후)
  - 잔금: 600만원 (전체 개발 완료 후)

## 기술적 특이사항

- 등기부등본 PDF 500~3,000개 동시 처리
- 배치 처리 및 멀티프로세싱 최적화
- 오류 처리 및 재처리 기능

### 복잡한 데이터 구조
- 엑셀 데이터 디스크 ↔ 등기부등본 자동 매칭
- 다중 시트 데이터 통합 ("비핵심 전체" 형태)
- 계산 수식 자동 적용

## 데이터 흐름

### 입력 데이터
1. **엑셀 데이터 디스크**: 채권 기초 데이터 (일괄 업로드, 자동 컬럼 매핑)
2. **등기부등본 PDF**: 500~3,000개 대량 업로드 (Supabase Storage)

### 처리 과정
1. **OCR 처리**: Python OCR 코드로 PDF에서 데이터 추출
2. **자동 매칭**: 데이터 디스크 ↔ 등기부등본 데이터 매칭
3. **데이터 편집**: 실시간 수정, 병합, 삭제
4. **계산 수식 적용**: 복잡한 계산 수식 자동 적용

### 출력 데이터
1. **Excel 파일**: 다중 시트, 수식 적용, 스타일 적용
2. **지도 시각화**: 물건 위치, 매매가, 토지/건물 정보
3. **통계 분석**: 차트 및 Excel 출력

## 화면 구성

### 주요 화면
1. **물건 목록 화면**: 검색, 필터, 정렬, 페이지네이션
2. **물건 상세 화면** (탭 구조):
   - 기초 데이터 탭
   - 등기부 정보 탭
   - 권리 분석 탭
   - 평가 시트 탭
   - 지도 탭
3. **대시보드**: 통계 카드, 차트, 빠른 작업
4. **관리자 화면**: 사용자 관리, 설정 관리, 시스템 로그
